One-page cheat sheet.

60 +5 Q · 105 min · 63% to pass
Scenario-based · lowest-code answer wins

01 Fundamentals 23%

  • Declarative first; code only for callouts, complex bulk, bespoke UI, async.
  • Object/Field = Profiles + Perm Sets (perm sets only add).
  • Record = OWD → role → sharing rules → manual → Apex (opens up only).
  • Reports: Tabular/Summary/Matrix/Joined — charts need grouping.
  • Report type "must have" vs "may or may not have" related.
  • Dashboard shows running user's data; dynamic = own data.
  • Compact layout = mobile key fields; feed tracking ≠ field history.

02 Data Modeling 22%

  • Record types = different picklists / layouts / process, per profile.
  • Master-Detail (ROCS): Roll-ups, Owner inherited, Cascade, Sharing by parent; max 2/object.
  • Lookup: loose, optional, own sharing, no roll-up, reparent freely.
  • Many-to-many = junction w/ 2 master-details; 1st = primary.
  • External ID → upsert / match key.
  • Long Text→Text truncates; many type changes wipe data — sandbox test.
  • Import Wizard ≤50k; Data Loader ≤5M + upsert; live external → Salesforce Connect.

03 Business Logic 28%

  • Formula: read-only, up to parent, never children.
  • Roll-up: COUNT/SUM/MIN/MAX, master-detail only.
  • Validation: formula TRUE = block; fires on API/import too; exempt via $Profile.
  • Approval process = only formal sign-off + record lock.
  • New automation = Flow; same-record = before-save; related/email = after-save.
  • Know order of execution; avoid recursion (gate w/ ISCHANGED).

04 User Interface 17%

  • App Builder builds record / app / home pages; visibility rules.
  • Page layout = fields, related lists, actions; Lightning page = arrangement.
  • Dynamic Forms = per-field placement + visibility (no code).
  • JS buttons don't work in Lightning → action / Flow / LWC.
  • Object-specific action pre-fills from parent; global doesn't.
  • Embed a screen Flow via Flow component for guided UI.
  • Custom LWC only when nothing standard fits; needs correct targets.

05 App Deployment 10%

  • Sandboxes: Dev & Dev Pro = metadata, 1-day; Partial = sample, 5-day; Full = all data, 29-day.
  • Change sets: related orgs only, no deletes, metadata only, add dependencies.
  • Managed = upgradeable / namespaced / IP-protected; Unmanaged = editable template.
  • Unrelated orgs / customers → package; deletions → Metadata API.
  • Apex to production → ≥75% coverage.

★ Tie-breakers

  • Two valid answers → pick less code / more scalable.
  • Aggregate children → roll-up (M-D), or Flow if lookup.
  • Block a bad save → validation rule.
  • Formal approval → approval process, not Flow.
  • Fast same-record update → before-save Flow.
  • Realistic-but-not-full data → Partial Copy sandbox.
  • Reach an unrelated org → package, not change set.
  • Show one field conditionally → Dynamic Forms.
