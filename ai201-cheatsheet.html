D1

AI Agents

35%
  • Hierarchy: Agent → Topics → Actions; Atlas routes.
  • Actions: standard, or custom via Flow / Apex / Prompt.
  • Types: Service, Sales/SDR, Employee, custom — sort by who it serves + auth.
  • Agent User = the running user; its perms bound all access.
  • Determinism: variables + guard filters. Var must be mutable & updated at runtime.
  • Channels: Messaging, email, Slack, Agent API.
Debug order
Topic routing → Action selection → Action logic
D2

Prompt Engineering

20%
  • Author reusable templates in Prompt Builder.
  • Ground to kill hallucination — grounding beats re-wording.

·

Einstein Trust Layer

  • Secure retrieval — respects Agent User perms.
  • Dynamic grounding — trusted data at runtime.
  • Data masking — PII tokenised, then restored.
  • Zero retention — provider stores/ trains on nothing.
  • Toxicity + audit — scored & logged.
Answer map PII → masking  ·  "not used to train" → zero retention  ·  logging → audit trail.

  • Chunking: larger = context, smaller = precision; semantic splits on meaning.
  • Retrievers: individual (one index) vs ensemble (blended).

D4

Deployment Lifecycle

20%
Build → Test → Deploy → Monitor ↺
  • Testing Center: batch utterances validate topic + action + response at scale.
  • Deploy sandbox → prod via Change Sets or Metadata API. Never build in prod.
  • Monitor: adoption dashboards, event logs, CSAT / NPS.
"Works in sandbox, not prod" Missing Agent User permission set, or inactive Search Index.
